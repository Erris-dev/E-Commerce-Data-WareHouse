## đź“ť General Principles

* **Case Style:** Use `snake_case` (all lowercase letters with underscores to separate words).
* **Language:** All object names must be in **English**.
* **Reserved Words:** Avoid using SQL reserved keywords (e.g., `SELECT`, `TABLE`, `ORDER`) as object names.
* **Clarity:** Names should be descriptive enough to convey the purpose of the object without referring to external documentation.

## đźŹ—ď¸Ź Table Naming Conventions

### Bronze Rules
The Bronze layer stores raw data. Tables must retain their original structure and naming from the source system to ensure traceability.
* **Pattern:** `<prefix>_<entity>`
* **`<prefix>`:** Functional group from source (e.g., `ent`, `ord`, `ref`).
* **`<entity>`:** Exact table name from the source.
* **Example:** `ent_olist_customers`

### Silver Rules
The Silver layer stores cleansed and standardized data. Like Bronze, these tables follow the source naming to maintain a direct lineage.
* **Pattern:** `<prefix>_<entity>`
* **Example:** `ord_olist_order_items`

### Gold Rules
The Gold layer uses business-aligned names optimized for reporting and analytics.
* **Pattern:** `<category>_<entity>`
* **`<category>`:** Describes the role (e.g., `dim` or `fact`).
* **`<entity>`:** Descriptive business name (e.g., `customers`, `sales`).
* **Example:** `dim_products`, `fact_sales`

#### Glossary of Category Patterns
| Pattern | Meaning | Example(s) |
| :--- | :--- | :--- |
| `dim_` | Dimension Table (Attributes) | `dim_customers`, `dim_products` |
| `fact_` | Fact Table (Metrics) | `fact_sales`, `fact_reviews` |
| `report_` | Specialized Report View | `report_monthly_revenue` |

## đź”‘ Column Naming Conventions

### Surrogate Keys
To decouple the Data Warehouse from source system changes, all primary keys in the Gold layer must use a surrogate key with a specific suffix.
* **Pattern:** `<entity>_key`
* **Example:** `customer_key`, `product_key`

### Technical Columns
Metadata columns generated by the system for auditing and ETL tracking.
* **Pattern:** `dwh_<column_name>`
* **`dwh`:** Prefix indicating system-generated metadata.
* **Example:** `dwh_load_date`, `dwh_active_flag`

## âš™ď¸Ź Stored Procedures

All stored procedures used for the ETL/ELT process must indicate the target layer they are processing.
* **Pattern:** `load_<layer>`
* **`<layer>`:** The specific Medallion layer being populated.
* **Example:** * `load_bronze` (Initial Ingestion)
    * `load_silver` (Transformation & Cleaning)
    * `load_gold` (Star Schema Generation)
